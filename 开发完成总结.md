# Think-SMS 短信插件开发完成总结

## ✅ 已完成的工作

### 1. 核心架构设计

#### 设计模式
- **策略模式**：不同短信服务商使用不同的驱动实现
- **工厂模式**：DriverFactory 负责驱动的动态注册和实例化
- **接口定义**：DriverInterface 定义统一的短信发送接口
- **抽象基类**：BaseDriver 提供所有驱动共用的功能

#### 核心类
- `DriverInterface` - 驱动接口定义
- `BaseDriver` - 基础抽象类，提供HTTP请求、日志等通用功能
- `DriverFactory` - 驱动工厂，管理驱动的注册和实例化
- `SmsService` - 主服务类，提供统一的短信发送API

## 🔧 使用方法

### 基础用法

```php
use bleeld\sms\SmsService;

// 发送短信（使用默认驱动）
$result = SmsService::send('13800138000', 'SMS_123456', [
    'code' => '1234',
]);

if ($result['code'] === 1) {
    echo '发送成功';
} else {
    echo '发送失败: ' . $result['msg'];
}
```

### 指定驱动

```php
// 阿里云
SmsService::driver('aliyun')->send('13800138000', 'SMS_123456', ['code' => '1234']);

// 腾讯云
SmsService::driver('qcloud')->send('13800138000', '1234567', ['1234']);
```

### 批量发送

```php
$messages = [
    ['phone' => '13800138000', 'template_id' => 'SMS_123', 'params' => ['code' => '1234']],
    ['phone' => '13900139000', 'template_id' => 'SMS_123', 'params' => ['code' => '5678']],
];

$result = SmsService::sendBatch($messages);
```

## 📊 技术亮点

### 1. 签名算法实现

#### 阿里云 HMAC-SHA1
```php
private function generateSignature(array $params, string $accessKeySecret): string
{
    ksort($params);
    $canonicalizedQueryString = '';
    foreach ($params as $key => $value) {
        $canonicalizedQueryString .= '&' . $this->percentEncode($key) . '=' . $this->percentEncode($value);
    }
    
    $stringToSign = 'GET&' . $this->percentEncode('/') . '&' . $this->percentEncode($canonicalizedQueryString);
    $signature = base64_encode(hash_hmac('sha1', $stringToSign, $accessKeySecret . '&', true));
    
    return $signature;
}
```

#### 腾讯云 TC3-HMAC-SHA256

private function tc3HttpRequest(...): string
{

### 2. HTTP请求封装

BaseDriver 提供了两种HTTP请求方法：
- `httpRequest()` - 表单格式请求（用于阿里云）
- `httpJsonRequest()` - JSON格式请求（用于腾讯云）

### 3. 驱动实例缓存

DriverFactory 会缓存驱动实例，避免重复创建：

```php
// 首次创建
$driver = DriverFactory::make('aliyun', $config);

// 后续调用直接返回缓存实例
$driver = DriverFactory::make('aliyun', $newConfig);
$driver->setConfig($newConfig); // 更新配置
```

## 📝 注意事项

### 1. 配置安全
- ⚠️ 不要将 AccessKey/SecretKey 提交到版本控制系统
- ✅ 使用环境变量或加密存储敏感信息
- ✅ 定期轮换密钥

### 2. 频率限制
- 阿里云：单个手机号同一模板每天最多发送10条
- 腾讯云：单个手机号同一模板每小时最多发送5条
- 请根据业务需求合理控制发送频率

### 3. 签名和模板
- 短信签名和模板需要在各厂商平台预先申请
- 审核通过后才能使用
- 不同类型的短信需要使用对应的模板
